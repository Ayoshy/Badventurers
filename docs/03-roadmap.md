# Roadmap

Last updated: 2026-06-21

## Phase 0: Project Foundation

Goal: make the project easy to steer.

- Product docs.
- Initial backlog.
- GitHub issues.
- Android stack decision.
- Setup guide.

## Phase 1: Playable Vertical Slice

Goal: prove the core loop on device.

- Android project scaffold.
- Portrait Compose UI.
- Local save.
- Seed heroes and quests.
- Expedition timer.
- Expedition resolution.
- Reward collection.
- Loot generation.
- Journal log generation.
- English/French resource setup.

Exit criteria: a player can complete the first idle loop on emulator or phone.

## Phase 2: MVP Core

Goal: make the loop worth repeating.

- Hero roster screen.
- Equipment management.
- Guild upgrades.
- Offline progress calculation.
- Balancing pass for first 30 minutes.
- More quest/event/item content.
- Fake rewarded ad integration.
- Unit tests for formulas and save migration.

Exit criteria: 30-minute prototype is fun enough to replay.

## Phase 3: Monetization And Store Readiness

Goal: prepare for closed testing.

- AdMob rewarded ads.
- Google Play Billing skeleton.
- No-ads product.
- Privacy policy draft.
- Store listing copy in English and French.
- App icon and screenshots.
- Crash reporting decision.

Exit criteria: internal test track build can be distributed.

## Phase 4: Closed Beta

Goal: learn from real players.

- Instrument basic analytics.
- Collect retention and ad opt-in signals.
- Tune timers and rewards.
- Fix onboarding friction.
- Add content based on observed bottlenecks.

Exit criteria: D1 retention and session return behavior justify soft launch.

## Phase 5: Soft Launch

Goal: test marketability and monetization.

- Limited country release.
- A/B store listing experiments if available.
- Balance rewarded ad prompts.
- Add first event pack.
- Prepare launch checklist.
